# Browser Use 产品目标

## 定位

甘草 Copilot 的自动化能力正式以 **Browser Use** 为目标：用户描述希望在浏览器中完成的结果，代理负责理解目标、观察页面、制定计划、执行动作、校验结果、失败恢复并交付数据或文件。

Browser Use 不是系统级 Computer Use。它不控制桌面应用，也不以屏幕坐标模拟整台电脑；能力边界限定在用户授权的 Chrome 标签页、扩展 API 和本地资料库中。

## 核心闭环

```text
用户目标
→ TaskPlan 编译与风险判断
→ 观察标签页和页面语义集合
→ 规划下一批原子动作
→ 解析目标元素
→ 执行动作
→ 重新观察并校验
→ 失败恢复或请求用户确认
→ 交付页面结果、结构化数据或下载文件
```

## 核心能力

1. 标签页与导航：打开站点、切换或跟随新标签页、回退、刷新和等待页面稳定。
2. 页面理解：识别导航、表单、表格、列表、搜索结果、弹窗、文件和页面阻塞信号。
3. 交互执行：点击、输入、选择、按键、滚动、上传、下载和等待。
4. 数据提取：提取页面文本、表格、列表、字段和文件元数据。
5. 多阶段任务：跨页面保留变量、下载结果、阶段证据和恢复检查点。
6. 安全控制：提交、删除、保存、发送、支付等高风险动作必须确认。
7. 可观测性：每一步记录观察、计划、目标解析、动作、验收和失败原因。

## 协作能力

- 页面诊断为 Browser Use 提供错误、权限、登录和页面健康信号。
- OCR 与资料中心承接下载文件和页面材料，完成解析、索引和问答。
- Memory 提供用户确认过的偏好、术语和流程经验。
- 页面监控能够触发 Browser Use 任务，并通过插件通知或 Webhook 交付结果。
- 工作流可以保存成功任务，将动态代理能力沉淀为可复用流程。

## 架构原则

- LLM 决定语义目标和短期计划，不直接猜 CSS selector。
- TargetResolver 负责把语义目标解析为真实页面元素。
- 每个阶段必须有正向完成证据，禁止“未执行但显示完成”。
- 规则只用于安全约束、确定性信号和降级，不作为中文语义主解析器。
- 动态任务优先重新观察和重新规划，不固化临时 selector 或坐标。
- 浏览器权限、用户登录态和数据边界始终优先于自动完成任务。

## 兼容策略

现有内部类型和协议继续保留：

- `computer_use` 任务类型
- `ComputerUse*` TypeScript 类型
- `RUN_COMPUTER_USE`、`STOP_COMPUTER_USE` 和相关事件
- `computerTask` 工作流步骤

这些名称仅作为兼容层，避免破坏历史任务、IndexedDB、工作流和扩展消息。用户可见名称、新文档和新增模块统一使用 Browser Use。后续如需迁移内部协议，应增加版本化适配器，而不是直接批量重命名。

## 当前内核

Browser Use 内核在现有阶段 Runner 上补充了以下通用能力：

- `browserUseActionRegistry.ts` 统一声明原子动作、页面工具、风险等级和是否可能打开新标签页，Runner 不再维护分散的动作映射分支。
- `BrowserUseSession` 记录初始标签页、当前受控标签页和同窗口标签页；点击或按键打开直接子标签页后，任务会跟随新标签页继续观察和执行。
- `ComputerUseRunState` 增加 `browserSession` 和阶段 `outputs`，跨页面任务可以持续携带标签页状态、下载结果和阶段输出。
- 浏览器级原子动作覆盖 `open_tab / switch_tab / close_tab / go_back / go_forward / reload`；页面内动作仍由 content script 执行。
- 用户明确说“新页面/新标签页”时，`open_site` 阶段创建并接管新标签页；普通“打开”继续复用当前标签页。
- Planner 可以读取前序阶段输出，并通过 `{{phaseId.path}}`、`{{outputs.phaseId.path}}`、`{{download.filename}}` 和 `{{currentTab.title}}` 引用跨阶段变量。
- 新标签页创建后会等待加载完成再观察页面，避免在空白页或中间态上规划后续动作。

现有 `ComputerUseRunner` 仍是兼容执行器，但新增 Browser Use 能力通过独立模块接入，后续可以逐步替换内部协议而不破坏历史任务。

### 当前边界

- 标签页会话限定在任务启动时所在窗口，不会静默接管其他窗口。
- 阶段变量保存 JSON 兼容结果；复杂二进制结果通过资料 ID 或下载结果引用。
- 浏览器历史动作依赖 Chrome 对当前页面提供的历史记录；受限页面、已关闭页面和跨域 iframe 会明确阻塞。

## 版本完成标准

### Browser Use Core

- 搜索并进入第 N 个自然结果。
- 同名父子菜单能够选择正确路径。
- 表单字段按 label 与上下文填写并验证。
- 表格行与行内动作正确绑定。
- 真实下载能够捕获、关联并保存结果。
- 登录、验证码、权限不足和受限页面能够明确阻塞。

### Browser Use Business Loop

- 支持“进入页面 → 筛选 → 查询 → 操作指定行 → 下载 → 资料入库 → 生成摘要”。
- 任务失败后可从检查点恢复，不重复已完成阶段。
- 关键业务 E2E 连续执行成功率达到 90% 以上。
- 所有失败都提供当前阶段、目标、最后页面、候选和恢复建议。
